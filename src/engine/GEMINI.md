# Game Engine Implementation Guide

## 1. 브라우저 호환성
- Node.js 전용 모듈(`fs`, `path` 등) 사용 금지. 브라우저에서 독립적으로 실행 가능한 순수 로직으로 작성.
- 상태 유지가 필요한 경우 `sessionStorage` 또는 `localStorage`를 활용하여 세션 지속성 확보.

## 2. 코어 상태 관리 및 페이즈 (추가 룰 반영)
- `INIT`: 게임 초기화. **1세트 선공은 주사위(50:50 확률 난수)로 결정. 2~4세트는 이전 세트까지의 누적 점수가 높은 플레이어가 선공 할당.** (동점 시 기존 선공 유지).
- `DRAFTING`: 타일 선택 페이즈. **선공부터 1장씩 번갈아가며 픽을 진행(총 10픽).** 플레이어가 픽한 타일의 색상 정보는 즉시 게임 상태(State)에 공개 기록되어 양측이 인지할 수 있도록 처리. (각자 빨강 3, 검정 2 제한 강제).
- `BATTLE`: 대결 페이즈 (비동기 순차 제출).
  - **[신규 룰] 세트 내 5번의 라운드(대결)가 진행되는 동안, 매 라운드마다 제출 선공이 교대로 바뀜.** (예: 1, 3, 5라운드는 '세트 선공'이 먼저 제출하고, 2, 4라운드는 '상대방'이 먼저 제출).
  - **라운드 선공**이 먼저 타일을 1장 제출하여 상태를 업데이트함 (중앙에 뒷면 색상 렌더링).
  - **라운드 후공**은 선공이 제출한 타일의 색상 정보를 참조하여 자신의 타일을 제출함.
- `SCORING`: 라운드/세트 점수 정산. 승패 판정 로직 순서 엄수: 더블승 -> 역전승 -> 기본승.
- `END`: 4세트 종료 후 최종 승자 판정 및 연장전 진입 여부 결정.
