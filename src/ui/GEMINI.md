# Web UI Implementation Guide

## 1. 뷰 렌더링 및 레이아웃
- 모바일과 데스크톱 모두에서 플레이 가능하도록 반응형 웹 디자인(Responsive Web Design) 적용.
- 타일의 색상(빨강/검정)을 시각적으로 명확히 구분할 수 있도록 CSS 디자인 구성 (카드 형태의 UI).
- 자신의 타일은 앞면(숫자)이 보이게, 상대방(AI)의 타일은 뒷면(색상)만 보이게 렌더링.

## 2. 인터랙션 및 애니메이션 (추가 룰 반영)
- **선공 결정 연출**: 1세트 시작 시 주사위 굴림(또는 동전 던지기) 애니메이션이나 토스트 메시지로 선공을 명확히 표시. 2~4세트 시작 전에는 점수 비교를 통한 선공 변경 안내.
- **교대 픽(Drafting) UI**:
  - 선공부터 번갈아가며 1장씩 타일을 선택하도록 턴에 맞춰 UI 조작을 활성/비활성화.
  - 픽이 완료된 카드는 즉시 보드(바닥) 영역에 렌더링되어, 양측이 가져간 카드의 '색상'을 직관적으로 확인할 수 있게 구현.
- **비동기 대결(Battle) UI**:
  - 선공의 턴일 때 후공의 조작 영역을 차단(Disabled).
  - 선공이 카드를 제출하면, 중앙 대결 영역에 해당 카드의 **뒷면(색상)을 먼저 렌더링**하여 후공이 인지할 수 있도록 함.
  - 후공이 그 색상을 확인한 후 자신의 카드를 제출할 수 있도록 조작 영역 활성화.
  - 양측 제출이 완료되면 동시 공개(카드 뒤집기) 딜레이 애니메이션을 적용하여 긴장감 연출.

## 3. 정보 시각화
- 현재 세트, 진행 라운드, 양측의 누적 점수, 남은 타일의 색상 분포를 화면 상단에 상시 표기.
- 승패 판정 결과(기본승/역전승/더블승)를 토스트 메시지나 모달 팝업으로 즉각 피드백.
