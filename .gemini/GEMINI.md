# Doubles Plan Project Guidelines

## 1. 프로젝트 목표
- 1인용 심리 전략 카드 게임 <더블스 플랜>의 웹 브라우저 실행 환경 구축.
- GitHub Pages를 통한 정적 웹 사이트 배포 및 서비스.

## 2. 기술 스택
- 순수 Vanilla JavaScript (ES6+), HTML5, CSS3 사용 (필요시 경량 UI 프레임워크 적용).
- 서버사이드 렌더링이나 백엔드 연동 없이 브라우저 내에서 100% 로컬 로직으로 동작하도록 구현.

## 3. 핵심 게임 규칙 (구현 필수 사항)
- **컴포넌트**: 1~12 숫자 타일 (빨강: 3~9 / 검정: 1, 2, 10, 11, 12).
- **게임 구성**: 총 4세트 진행 (세트당 '블록 선택' 1회 + '숫자 대결' 5회).
- **선공 결정 (Turn Order)**:
  - 1세트: 주사위(랜덤)를 통해 첫 선공 플레이어를 결정.
  - 2~4세트: 이전 세트까지의 누적 점수가 더 높은 플레이어가 선공을 가져감. (동점일 경우 이전 세트 선공 유지).
- **블록 선택 (Alternating Draft)**:
  - 선공 플레이어부터 **번갈아가며 1장씩** 타일을 선택함.
  - 각자 빨강 3개, 검정 2개로 총 5개의 블록을 선택하며, **선택된 카드는 바닥(Board)에 뒷면(색상)이 공개되어 누구나 색상 구성을 인식**할 수 있음.
  - 남은 2종류의 블록은 비공개로 제외.
- **숫자 대결 (Battle Phase)**:
  - **선공이 먼저 카드를 1장 제출**하여 중앙에 뒷면 상태로 배치함.
  - **후공은 선공이 제출한 카드의 뒷면(색상)을 확인한 후**, 자신이 제출할 카드를 선택함.
  - 양측 모두 제출이 완료되면 동시 공개.
- **승패 판정**:
  - 기본승: 높은 숫자 승리
  - 역전승: (높은 숫자) > (낮은 숫자) × 2 인 경우, **낮은 숫자 승리**
  - 더블승: (높은 숫자) == (낮은 숫자) × 2 인 경우, **높은 숫자 승리**
- **점수 산정**:
  - 1세트: 모든 승리 1점
  - 2세트: 기본/역전승 1점, 더블승 2점
  - 3, 4세트: 기본/역전승 2점, 더블승 4점
- **승리 조건**: 4세트 종료 후 합산 점수가 높은 쪽 승리. 동점 시 3/4세트 룰로 연장전.
